# s3 connector (`s3://` — S3 / R2 / GCS / MinIO)

An S3-compatible bucket's object-key tree, keys kept verbatim as paths
(`/app/2026-05-10/app.jsonl`, `/reports/Q1.csv`, `/images/header.png`). Works
against R2 / GCS / MinIO via `endpoint_url`.

**object_kind by extension** (same mapping as the file connector):
- code (`.py/.js/.go/...`) → code; `.md/.pdf/.docx/.html` → document (pdf/docx
  auto-converted to markdown); images → VLM description; `.json/.csv/.log/...`
  → text_blob (grep works, not embedded by default); other → binary.

**cat**: text as-is; converted-doc → markdown; image → VLM description; binary →
`<binary, N bytes>` (use `--range`/`export` for big objects).

**workflow on a large bucket**: `mfs search "<intent>" s3://<alias>/<prefix>` →
hit has `lines` → `mfs cat <key> --range start:end`. `mfs grep` for literal keys/ids.
Rename/fingerprint via ETag. Config: `bucket`, `prefix`, `region`,
`access_key_id`/`secret_access_key`, optional `endpoint_url`.
