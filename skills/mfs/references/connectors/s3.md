# s3 connector (`s3://` — S3 / R2 / GCS / MinIO)

## What this is

Any S3-compatible object store: AWS S3, Cloudflare R2, Google Cloud Storage
(via its S3-compatible interop endpoint), MinIO, Backblaze B2. The
connector uses `aioboto3` (async wrapper over boto3) and supports
`endpoint_url` to point at non-AWS endpoints.

Each S3 key is exposed verbatim as a file path. The mapping from extension
to **object_kind** is shared with the **file** connector — code / docs /
images / binary handled the same way.

**When MFS helps**: log buckets, document archives, scraped data dumps,
image collections — semantic + literal search across thousands of objects
without downloading them up front.

**Cost note**: every `cat` / `head` / `grep` does a `GetObject` (egress
billed by the cloud); sync does `ListObjectsV2` (small per-request cost) +
`GetObject` on changed keys. Scope with `prefix` to avoid scanning whole
buckets.

## URI shape

```
s3://my-bucket/                                connector root
s3://my-bucket/logs/2026-05-10/app.jsonl      one key (verbatim)
s3://my-bucket/reports/Q1.csv
s3://my-bucket/images/header.png
```

If `prefix` is set, the connector strips it from the displayed path — e.g.
`prefix = "reports/"` makes `Q1.csv` be at `s3://<alias>/Q1.csv`, not
`s3://<alias>/reports/Q1.csv`.

## Auth

Static IAM access keys are the supported path. The connector accepts the
two fields either inline (dev) or combined under `credential_ref` (prod):

```toml
# inline (works, but gets redacted on persistence → won't survive a restart):
# access_key_id     = "AKIAxxx..."
# secret_access_key = "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"

# preferred:
credential_ref = "env:S3_KEYS"      # value is "AKID:SECRET" (colon-joined)
# or
credential_ref = "file:/var/run/secrets/s3/keys.txt"   # one line "AKID:SECRET"
```

For R2 / GCS / MinIO: same fields, plus `endpoint_url`. No SigV2 / no
session tokens currently.

For AWS: temporary STS credentials work if you assemble them into the
`AKID:SECRET` form, but the connector doesn't currently support a session
token — for STS-issued creds use an IAM long-lived access key instead.

## Connector config TOML

```toml
# ─── bucket (required) ───
bucket = "my-bucket"
credential_ref = "env:S3_KEYS"   # value = "AKID:SECRET"

# ─── scope ───
# prefix = "logs/2026/"                     # restrict to this key prefix; default = whole bucket
# region = "us-east-1"                       # default us-east-1

# ─── non-AWS endpoints ───
# endpoint_url = "https://<accountid>.r2.cloudflarestorage.com"   # Cloudflare R2
# endpoint_url = "https://storage.googleapis.com"                 # GCS interop
# endpoint_url = "http://minio.internal:9000"                     # self-hosted MinIO

# ─── crawl bound ───
# max_objects = 100000                       # safety cap on ListObjectsV2 enumeration
```

No `[[objects]]` — S3 objects are indexed exactly like local files (text /
code / image / binary by extension).

## What each command does

| Command | Behaviour |
|---|---|
| `mfs ls s3://<alias>/<prefix>/` | virtual directory listing built from `ListObjectsV2` with `Delimiter='/'`. |
| `mfs cat s3://<alias>/<key>` | `GetObject` + decode (text/code) or convert (pdf/docx/html → markdown) or VLM-describe (image). |
| `mfs cat <key> --range A:B` | partial `GetObject` using HTTP `Range:` header (no full download). |
| `mfs head -n N <key>` | partial `GetObject` of the first ~N lines' worth of bytes (estimated). |
| `mfs grep "PATTERN" <prefix>` | scans matching keys; downloads each via `GetObject` (cost-aware: bound the prefix). |
| `mfs search "QUERY"` | Milvus only — no S3 call. |
| `mfs export <key>` | streams the raw object to stdout / file. |

## Typical workflow

```bash
# 1. IAM-side: create a user with s3:ListBucket + s3:GetObject on the prefix you'll index.
#    For R2/GCS: create access keys in the corresponding console.

# 2. Drop the AKID:SECRET into env or a file.
export S3_KEYS="AKIA...:wJalr..."

# 3. Register.
cat > prod-logs.toml <<'EOF'
bucket = "acme-prod-logs"
prefix = "app/2026/"
region = "us-east-1"
credential_ref = "env:S3_KEYS"
EOF
mfs add s3://prod-logs --config prod-logs.toml

# 4. Search.
mfs search "ssl handshake failure" --connector-uri s3://prod-logs --top-k 5
mfs cat s3://prod-logs/app/2026/05/10/edge-7.jsonl --range 1200:1240

# 5. Re-sync (re-lists; downloads only ETag-changed objects).
mfs add s3://prod-logs --no-full
```

## Incremental sync (ETag / LastModified)

Per-key fingerprint = `ETag | LastModified` from `ListObjectsV2`. Unchanged
keys are skipped (no `GetObject`). New keys are indexed; deleted keys
(present in last sync, absent in this listing) have their vectors purged.
`prefix`-scoped listings make this O(N changed) for incremental refresh.

## Gotchas

1. **`prefix` scope is the only cost lever.** A 10M-object bucket without
   a prefix takes forever to list; ALWAYS narrow with `prefix`. Use
   multiple connectors for multiple prefixes.
2. **`credential_ref` format is `AKID:SECRET`** (colon-joined). Easy to
   get wrong — verify with `echo "$S3_KEYS" | tr -c -d ':' | wc -c` (should
   print `1`).
3. **R2 / GCS / MinIO**: always need `endpoint_url`. AWS S3 leaves it
   blank.
4. **Range reads** (`cat --range`, `head -n N`) use HTTP Range headers and
   are efficient even for huge objects — favour them over full `cat` for
   big logs.
5. **Image indexing** burns VLM tokens per image. If your bucket has
   thousands of images, either lower `summary.include_image_desc`
   (server.toml) or filter image extensions in a future `.mfsignore`-style
   field (not yet supported on s3 — track per-key `indexable=false` via
   key naming conventions).
6. **No STS session token support** today. Use long-lived IAM keys
   (preferably narrow ones from a dedicated `mfs-reader` IAM user).
