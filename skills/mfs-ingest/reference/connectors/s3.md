# s3 connector — ingest

URI: `s3://<alias>`.

Works against AWS S3 and any S3-compatible store: Cloudflare R2, Google
Cloud Storage (with S3 interop), MinIO, Wasabi, Backblaze B2. The
`endpoint_url` field is what selects between them.

## How to obtain credentials

**AWS S3**:
- IAM user → **Security credentials → Access keys → Create access
  key**.
- Or use STS temporary credentials via the SDK (boto3 reads env vars
  `AWS_ACCESS_KEY_ID` + `AWS_SECRET_ACCESS_KEY` automatically; if those
  are set, you can leave them out of the toml).
- Recommended IAM policy:
  ```json
  {
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Action": ["s3:GetObject", "s3:ListBucket"],
      "Resource": ["arn:aws:s3:::my-bucket", "arn:aws:s3:::my-bucket/*"]
    }]
  }
  ```

**Cloudflare R2**:
- R2 dashboard → **Manage R2 API Tokens → Create API Token** with
  Object Read scope.
- `endpoint_url`: `https://<account-id>.r2.cloudflarestorage.com`.
- `region`: `auto`.

**Google Cloud Storage (S3 interop)**:
- IAM & Admin → Service Account → keys + enable S3 interop on the
  bucket.
- `endpoint_url`: `https://storage.googleapis.com`.

**MinIO (self-hosted)**:
- Service's access key + secret key.
- `endpoint_url`: your MinIO URL (e.g. `https://minio.internal:9000`).

## Required toml fields

| key | what |
|---|---|
| `bucket` | bucket name |
| `access_key_id` | (`env:AWS_ACCESS_KEY_ID` recommended) |
| `secret_access_key` | (`env:AWS_SECRET_ACCESS_KEY` recommended) |

## Optional

| key | default | meaning |
|---|---|---|
| `prefix` | (whole bucket) | key prefix to scope ingest, e.g. `docs/2026/` |
| `region` | `us-east-1` | AWS region (use `auto` for R2) |
| `endpoint_url` | (AWS) | set for R2 / GCS / MinIO |

## env: example (AWS S3)

```toml
bucket = "acme-docs"
prefix = "engineering/rfc/"
region = "us-west-2"
access_key_id = "env:AWS_ACCESS_KEY_ID"
secret_access_key = "env:AWS_SECRET_ACCESS_KEY"
```

```bash
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
mfs add s3://acme-docs --config /tmp/mfs-s3.toml
```

## env: example (R2)

```toml
bucket = "kb-archives"
prefix = ""
region = "auto"
endpoint_url = "https://abc123.r2.cloudflarestorage.com"
access_key_id = "env:R2_ACCESS_KEY"
secret_access_key = "env:R2_SECRET_KEY"
```

## Pitfalls

- **Bucket region matters**: 301 redirects from the wrong region cost
  one extra round-trip per request. Set `region` correctly.
- **`prefix` is exact**: must end with `/` for "directory" semantics
  (otherwise `docs` also matches `docs-archive`).
- **Large objects**: a single 1GB PDF gets converted via markitdown
  and may be heavy in memory. Set `max_file_bytes` in [[objects]] if
  problematic.
- **Server-side encryption**: SSE-S3 / SSE-KMS keys must be accessible
  to the IAM principal; otherwise `GetObject` 403s.
- **Versioned buckets**: only the latest version of each key is
  indexed; older versions are invisible.
